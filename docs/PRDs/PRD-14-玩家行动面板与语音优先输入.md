# PRD-14 - 玩家行动面板与语音优先输入

**版本**: 1.0  
**状态**: 待开发  
**定位**: 手机游戏中首页与行动输入

## 1. 背景

玩家首版使用手机竖屏，公共叙事看 Host 大屏。手机首页不能像桌面后台那样铺满信息，而要优先回答“我现在该做什么、我能怎么说、我的行动提交到哪了”。语音是主入口，文字和按钮是补充。

## 2. 目标

- 以行动面板作为玩家手机首页。
- 主入口为按住说话。
- 保留文字输入和战术按钮兜底。
- 用底部标签进入角色、物品、线索、日志。

## 3. 范围边界

**包含**: 当前目标、语音按钮、文字兜底、战术按钮入口、底部导航。  
**不包含**: 队伍聊天、桌面端完整布局、语音实时通话。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-14-1 | 作为玩家，我打开手机就能看到当前目标和行动入口。 | P0 |
| US-14-2 | 作为玩家，我想按住说话提交行动。 | P0 |
| US-14-3 | 作为玩家，语音不可用时我还能打字。 | P0 |

## 5. 功能需求

1. 手机首页默认展示行动面板。
2. 行动面板包含当前团队目标、个人目标摘要、语音按钮、最近回执。
3. 按住说话调用 PRD-08 录音与 STT 流程。
4. 文本输入作为麦克风失败或玩家主动选择时的兜底。
5. 战术按钮来自 `s2c_tactical_prompt`。
6. 底部标签包含行动、角色、物品、线索、日志。
7. actionState 非 IDLE 时输入入口禁用或显示等待状态。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| Intent | `voice_command` | 语音行动 |
| Intent | `dialogue` | 文字行动 |
| Event | `s2c_tactical_prompt` | 快捷行动 |
| Store | `PersonalObjective` | 个人目标摘要 |
| Store | `ActionReceipt` | 最近行动回执 |

## 7. 状态与错误处理

- 麦克风被拒时自动展开文字输入。
- 网络断开时输入入口置灰并提示正在重连。
- 当前目标为空时显示“等待 AI KP 开场”。
- 长文本输入超过限制时提示精简行动描述。

## 8. 验收标准

- 手机竖屏首屏能看到当前目标、语音按钮和行动状态。
- 语音、文字、按钮都走统一意图网关。
- 底部导航可进入角色/物品/线索/日志。
- actionState 非 IDLE 时不会重复提交。

## 9. 测试场景

1. 玩家进入游戏后默认落在行动面板。
2. 按住说话提交语音行动。
3. 禁用麦克风后出现文字输入。
4. 收到 tactical prompt 后显示行动按钮。

## 10. 风险依赖

- 依赖 PRD-08 的语音能力。
- 小屏幕上按钮和状态文案需要严格控制高度。
- 远程跑团若无大屏，后续需补公共叙事模式。
