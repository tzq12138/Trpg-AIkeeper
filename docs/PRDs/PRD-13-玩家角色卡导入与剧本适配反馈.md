# PRD-13 - 玩家角色卡导入与剧本适配反馈

**版本**: 1.0  
**状态**: 待开发  
**定位**: xlsx 角色卡导入与剧本标签匹配

## 1. 背景

首版允许玩家自带角色卡，不强制使用剧本预设角色。系统需要优先支持 xlsx 角色卡导入，并根据剧本推荐标签给出适配提醒：缺技能、缺职业背景或人数结构风险都提示，但不阻止玩家开始。

## 2. 目标

- 支持手机上传 xlsx 角色卡。
- 解析 COC 7e 角色核心字段。
- 生成 `CharacterCompatibilityReport`。
- 让玩家确认导入结果后进入 ready 流程。

## 3. 范围边界

**包含**: xlsx 上传、字段解析、预览确认、标签适配、风险提示。  
**不包含**: 图片/OCR/PDF 角色卡、完整角色编辑器、自动加点。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-13-1 | 作为玩家，我想上传已有 xlsx 角色卡直接开玩。 | P0 |
| US-13-2 | 作为玩家，我想看到系统是否正确解析了我的角色。 | P0 |
| US-13-3 | 作为玩家，我想知道我的角色和剧本是否匹配。 | P1 |

## 5. 功能需求

1. 玩家在准备页上传 `.xlsx`。
2. 服务端解析姓名、属性、HP、SAN、MP、Luck、技能、物品和背景摘要。
3. 手机展示解析预览，玩家确认后提交 `character_import_confirm`。
4. 系统根据剧本 `recommendedTags`、`requiredPlayerCount`、`keySkills` 生成适配报告。
5. 适配报告分为 `matched`、`warning`、`missing`。
6. 缺少推荐技能/背景时提示但允许 ready。
7. JSON/图片/PDF 角色卡在首版显示“暂不支持”。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| REST | `POST /api/player/character/import-xlsx` | 上传并解析 xlsx |
| Intent | `character_import_confirm` | 确认绑定角色 |
| Type | `CharacterCompatibilityReport` | 剧本适配报告 |
| Event | `s2c_full_snapshot` | 导入后同步角色状态 |

## 7. 状态与错误处理

- xlsx 解析失败时显示可读错误和重新上传入口。
- 关键字段缺失时允许手动确认“仍使用”，但标记高风险。
- 上传大文件超限时提示压缩或更换文件。
- 同一玩家重新导入角色会覆盖未开始房间中的角色绑定。

## 8. 验收标准

- xlsx 角色卡可解析并预览核心字段。
- 适配失败只提醒不阻止。
- 确认导入后 Player Store 能收到角色快照。
- 不支持格式有明确提示。

## 9. 测试场景

1. 上传合法 xlsx，展示角色姓名、HP、SAN 和技能。
2. 缺少关键技能时出现 warning。
3. 上传损坏 xlsx，提示解析失败。
4. 确认导入后 ready 流程解锁。

## 10. 风险依赖

- 依赖现有 COC xlsx 解析能力可复用。
- 不同角色卡模板字段差异可能导致解析失败。
- 剧本标签来自 PDF 结构化结果，质量受 PRD-22 影响。
