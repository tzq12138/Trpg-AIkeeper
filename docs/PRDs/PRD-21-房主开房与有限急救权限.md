# PRD-21 - 房主开房与有限急救权限

**版本**: 1.0  
**状态**: 待开发  
**定位**: AI 自动 KP 模式下的开桌管理员

## 1. 背景

房主可以作为玩家参与，所以不能拥有完整 KP 剧透视角。房主职责是创建房间、选择剧本、邀请玩家、启动 Host 大屏，并在 AI KP 卡住或明显误判时做有限急救。

## 2. 目标

- 支持房主创建房间和邀请玩家。
- 房主可选择已解析剧本并启动大屏。
- 提供暂停、重试 AI 回合、提交急救说明等薄权限。
- 防止房主看到完整真相、未发现线索和其他玩家私密内容。

## 3. 范围边界

**包含**: 开房、二维码邀请、剧本选择、Host 启动、暂停/继续/重试。  
**不包含**: 完整 KP 控制台、真相编辑、手动改状态。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-21-1 | 作为房主，我想快速创建房间并邀请朋友扫码加入。 | P0 |
| US-21-2 | 作为房主玩家，我不想因为开房看到剧透。 | P0 |
| US-21-3 | 作为房主，我需要在 AI 明显卡住时暂停或重试。 | P0 |

## 5. 功能需求

1. 房主创建房间时选择剧本和防剧透档位。
2. 系统生成房间码、玩家二维码、Host 大屏链接。
3. 房主可在大厅查看玩家 ready 状态。
4. 房主可暂停/继续 AI KP。
5. 房主可请求“重试上一次 AI 回合”。
6. 房主可提交不含剧透的急救说明。
7. 房主不能查看完整真相、隐藏线索、NPC 幕后动机。
8. 房主若也导入角色，则在 Player 侧作为普通玩家参与。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| REST | `POST /api/rooms` | 创建房间 |
| REST | `POST /api/rooms/:roomId/start` | 开始游戏 |
| Intent | `owner_pause` / `owner_retry_turn` | 有限急救 |
| Type | `RoomOwnerPermission` | 房主权限 |
| Event | `s2c_room_lobby_snapshot` | 大厅状态 |

## 7. 状态与错误处理

- 剧本质量报告为高风险时仍可开局，但需二次确认。
- AI KP 正在关键结算时暂停排队到安全点。
- 重试不会回滚任意事件，只重试最近可重算 AI 回合。
- 房主权限请求都写入事件日志。

## 8. 验收标准

- 房主可开房、邀请、启动大屏和开始游戏。
- 房主参与玩家时不获得 KP 剧透。
- 暂停/继续/重试可用且有日志。
- 房主不能直接改 HP/SAN/线索。

## 9. 测试场景

1. 房主创建房间并生成二维码。
2. 房主导入角色后作为玩家 ready。
3. 房主点击暂停，AI KP 停在安全点。
4. 房主尝试查看真相，被拒绝。

## 10. 风险依赖

- 依赖剧本质量报告判断开局风险。
- 急救权限过弱可能救不了复杂错误，过强会剧透。
- 后续活动模式可单独扩展“非玩家管理员”。
