# PRD-27 - 事件日志存档回放与检查点

**版本**: 1.0  
**状态**: 待开发  
**定位**: 完整回放 + 检查点恢复

## 1. 背景

用户希望完整回放存档，但首版不做任意事件回滚和分支时间线。因此系统采用事件日志 + 场景检查点：可回放公共演出，可从检查点恢复继续。

## 2. 目标

- 保存玩家行动、AI 输出、Engine 结算、投影事件和状态快照。
- 支持公共演出回放。
- 支持检查点恢复继续。
- 为个人档案和战役复盘提供数据源。

## 3. 范围边界

**包含**: `EventLogEntry`、`Checkpoint`、公共回放、检查点恢复。  
**不包含**: 任意事件回滚、分支时间线、视频录制。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-27-1 | 作为玩家，我想回看关键公共演出。 | P1 |
| US-27-2 | 作为房主，我想中途暂停后下次继续。 | P0 |
| US-27-3 | 作为系统，我需要用日志支撑复盘查询。 | P0 |

## 5. 功能需求

1. 所有权威状态变更写入事件日志。
2. Host 投影事件记录为可回放事件。
3. 每个场景开始/结束创建检查点。
4. 手动暂停可创建检查点。
5. 恢复时从检查点快照加载状态，并继续接收新事件。
6. 回放只读，不改变当前房间状态。
7. 个人私密事件按权限过滤查询。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| Type | `EventLogEntry` | 事件日志 |
| Type | `Checkpoint` | 恢复点 |
| REST | `GET /api/rooms/:roomId/replay` | 公共回放 |
| REST | `POST /api/rooms/:roomId/restore/:checkpointId` | 检查点恢复 |

## 7. 状态与错误处理

- 检查点损坏时回退到上一个可用检查点。
- 回放期间不允许提交行动。
- 恢复后生成新的连续事件序列，不做分支。
- 私密事件回放必须按玩家权限过滤。

## 8. 验收标准

- 游戏过程关键事件都能进入日志。
- 公共 Host 演出可按时间线回放。
- 可从检查点恢复继续。
- 不支持任意事件回滚，界面明确边界。

## 9. 测试场景

1. 场景结束生成检查点。
2. 中途暂停后恢复房间。
3. 回放公共骰子和字幕事件。
4. Player B 查询不到 Player A 私密事件。

## 10. 风险依赖

- 依赖事件信封和序列号稳定。
- 长团日志可能膨胀，需要归档策略。
- 检查点频率影响恢复粒度和存储成本。
