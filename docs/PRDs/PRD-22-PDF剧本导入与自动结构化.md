# PRD-22 - PDF 剧本导入与自动结构化

**版本**: 1.0  
**状态**: 待开发  
**定位**: 从文字 PDF 自动生成 AI 可主持剧本结构

## 1. 背景

用户希望新开剧本时尽量只导入 PDF，后续由 AI 或脚本自动处理，降低准备门槛。首版优先支持可复制文字 PDF；扫描版/OCR 后置。

## 2. 目标

- 上传文字 PDF 并抽取文本。
- 自动拆分场景、NPC、线索、真相、结局、推荐角色标签。
- 保留原文引用，方便质量报告和主持时追溯。
- 本地优先保存剧本原文和结构化结果。

## 3. 范围边界

**包含**: PDF 文本抽取、章节切片、AI 结构化、原文引用、导入任务状态。  
**不包含**: 内置 OCR、云端剧本托管、人工完整编辑器。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-22-1 | 作为准备者，我想上传剧本 PDF 后自动得到可开局内容。 | P0 |
| US-22-2 | 作为准备者，我想知道哪些内容来自 PDF 哪一页。 | P0 |
| US-22-3 | 作为系统，我需要把真相和玩家可见信息分开。 | P0 |

## 5. 功能需求

1. 支持上传文字版 PDF。
2. 系统检测扫描 PDF，提示暂不支持或需外部 OCR。
3. 抽取文本后按页码、标题、段落切片。
4. AI/脚本抽取 `ScenarioKnowledgeGraph`。
5. 结构化内容至少包含场景、NPC、线索、真相、结局、危险点、推荐标签。
6. 每个结构化节点保留 source page 和 source excerpt。
7. 导入过程以 `ScenarioImportJob` 跟踪状态。
8. 解析结果进入 PRD-23 质量报告。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| REST | `POST /api/scenarios/import-pdf` | 上传 PDF |
| REST | `GET /api/scenarios/import-jobs/:jobId` | 查询导入状态 |
| Type | `ScenarioImportJob` | 导入任务 |
| Type | `ScenarioKnowledgeGraph` | 结构化剧本 |

## 7. 状态与错误处理

- PDF 无可抽取文本时标记为 `requires_ocr`。
- AI 结构化失败时保留文本切片并允许重试。
- 大 PDF 分批处理，避免单次模型上下文过长。
- 原文只本地保存；云 AI 只接收当前必要片段。

## 8. 验收标准

- 文字 PDF 可生成结构化剧本初稿。
- 扫描 PDF 有明确提示，不静默失败。
- 每个关键节点可追溯原文来源。
- 导入任务有进度和失败原因。

## 9. 测试场景

1. 上传文字 PDF，抽取场景/NPC/线索。
2. 上传扫描 PDF，返回 requires_ocr。
3. AI 解析中断后重试成功。
4. 查看线索节点能看到来源页码。

## 10. 风险依赖

- PDF 版式差异会影响章节识别。
- 云 AI 成本和隐私需要控制片段发送。
- OCR 后续可作为独立能力补充。
