# PRD-26 - 防剧透策略与暴露度控制

**版本**: 1.0  
**状态**: 待开发  
**定位**: AI 自动 KP 的信息边界与线索释放策略

## 1. 背景

AI 自动 KP 不能把完整剧本真相直接放进玩家上下文。用户希望防剧透可配置，首版采用三档：严谨、标准、电影化；默认标准偏严。

## 2. 目标

- 控制 AI KP 每轮可见信息。
- 根据暴露度释放线索和真相。
- 支持三档防剧透配置。
- 防止未发现线索、幕后动机、结局提前泄漏。

## 3. 范围边界

**包含**: `SpoilerProfile`、暴露度、上下文过滤、释放审计。  
**不包含**: 人工逐句审稿、完全防止玩家线下互相剧透。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-26-1 | 作为玩家，我不希望 AI 提前说出真相。 | P0 |
| US-26-2 | 作为准备者，我想按剧本风格选择剧透严格度。 | P0 |
| US-26-3 | 作为系统，我要记录每次线索释放依据。 | P1 |

## 5. 功能需求

1. 三档配置：`strict`、`standard`、`cinematic`。
2. 默认 `standard` 偏严。
3. 暴露度根据已发现线索、场景进度、玩家推理更新。
4. AI KP 上下文按 audience 和 visibility 过滤。
5. 未发现线索默认不可进入玩家可见叙事。
6. 真相和结局只在高暴露度或结局阶段释放。
7. 每次释放线索写入事件日志。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| Type | `SpoilerProfile` | 剧透档位 |
| Type | `ScenarioKnowledgeGraph` | 信息图谱 |
| Type | `ExposureState` | 暴露度 |
| Event | `s2c_private_notice` | 私密线索 |
| Event | `s2c_public_observation` | 公共观察 |

## 7. 状态与错误处理

- 无法判断权限的信息默认不释放。
- cinematic 档也不能直接泄漏结局关键答案。
- AI 输出含未授权真相时 Engine 拦截并重试。
- 暴露度异常下降时保留审计，不回收已知信息。

## 8. 验收标准

- 三档配置可选且默认 standard。
- 未发现关键线索不会进入玩家上下文。
- 线索释放有事件日志。
- AI 越界输出会被拦截。

## 9. 测试场景

1. strict 档下未发现线索不进入提示词。
2. cinematic 档下允许氛围性暗示但不说结局。
3. 玩家发现关键线索后暴露度上升。
4. AI 输出幕后真相，被 Engine 拦截。

## 10. 风险依赖

- 依赖 PDF 结构化正确标记真相和线索。
- 防剧透过严可能导致 AI 叙事贫乏。
- 暴露度算法需要持续调优。
