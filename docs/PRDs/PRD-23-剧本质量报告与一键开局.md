# PRD-23 - 剧本质量报告与一键开局

**版本**: 1.0  
**状态**: 待开发  
**定位**: PDF 自动结构化后的开局风险评估

## 1. 背景

准备者不想逐项编辑剧本，但自动解析可能漏掉真相、结局、关键线索或防剧透边界。系统需要给质量报告和一键开局能力：能开就开，有风险就明确提示。

## 2. 目标

- 自动评估剧本是否足够开局。
- 报告关键缺口和风险等级。
- 允许准备者一键创建可玩房间。
- 支持高风险二次确认，而不是强制人工审核。

## 3. 范围边界

**包含**: 完整性检查、剧透风险、角色适配建议、开局按钮。  
**不包含**: 完整剧本编辑器、人工审稿流程、自动修复所有问题。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-23-1 | 作为准备者，我想知道这个 PDF 解析后能不能直接开局。 | P0 |
| US-23-2 | 作为房主，我想看到风险但仍能一键开房。 | P0 |
| US-23-3 | 作为系统，我要提示可能剧透或缺关键线索的位置。 | P0 |

## 5. 功能需求

1. 生成 `ScenarioQualityReport`。
2. 报告包含完整性、可主持性、防剧透、角色适配、素材缺口。
3. 风险等级为 `ready | warning | highRisk | blocked`。
4. `blocked` 仅用于无法抽取文本或无结构化结果。
5. `warning/highRisk` 可二次确认后开局。
6. 一键开局创建房间并绑定剧本结构。
7. 报告保留建议修正项，但不要求首版提供完整编辑。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| REST | `GET /api/scenarios/:scenarioId/quality-report` | 查询质量报告 |
| REST | `POST /api/scenarios/:scenarioId/create-room` | 一键开局 |
| Type | `ScenarioQualityReport` | 质量报告 |
| Type | `SpoilerProfile` | 防剧透档位 |

## 7. 状态与错误处理

- 报告生成失败时不允许一键开局。
- highRisk 开局需房主确认。
- 推荐人数与实际玩家不符时提示但允许开始。
- 防剧透配置缺失时使用默认标准偏严档。

## 8. 验收标准

- 每个导入剧本都有质量报告。
- 文字 PDF 正常解析后可一键创建房间。
- 高风险项可见且不被隐藏。
- blocked 剧本不能开局。

## 9. 测试场景

1. 完整剧本报告为 ready，可开房。
2. 缺结局报告 warning，可确认开房。
3. 扫描 PDF 报 blocked。
4. 人数不匹配显示风险但允许继续。

## 10. 风险依赖

- 报告准确性依赖 PRD-22 结构化质量。
- 过多风险提示可能吓退用户，需要文案分层。
- 一键开局不能绕过基础协议初始化。
