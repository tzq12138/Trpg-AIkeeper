# PRD-17 - 玩家个人目标与当前任务

**版本**: 1.0  
**状态**: 待开发  
**定位**: 团队目标、个人目标和当前可行动提示

## 1. 背景

AI 自动 KP 没有人类主持人持续提醒玩家“现在该干嘛”。手机行动面板需要展示团队当前目标；如果剧本解析出个人动机或秘密目标，则向对应玩家显示个人目标。个人目标是可选能力，不能要求每个剧本都有。

## 2. 目标

- 展示当前团队目标和场景任务。
- 可选展示玩家个人目标。
- 根据剧情进度更新目标状态。
- 避免个人目标泄漏给其他玩家或 Host 大屏。

## 3. 范围边界

**包含**: 当前目标、个人目标、完成/失败/隐藏状态。  
**不包含**: 强制剧本杀阵营系统、积分排行榜、PvP 规则。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-17-1 | 作为玩家，我想知道当前团队正在调查什么。 | P0 |
| US-17-2 | 作为玩家，如果我有个人目标，我希望只在自己手机看到。 | P1 |
| US-17-3 | 作为 AI KP，我需要根据目标引导玩家行动。 | P0 |

## 5. 功能需求

1. 行动面板展示 `currentTeamObjective`。
2. 有个人目标时展示 `PersonalObjective` 卡片。
3. 个人目标包含标题、描述、状态、可选提示。
4. 目标更新由 Engine 下发 `s2c_state_patch`。
5. 没有个人目标时不显示空模块。
6. 团队目标可在 Host 大屏显示，个人目标不得显示。
7. 目标完成后进入个人档案。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| Type | `PersonalObjective` | 个人目标 |
| Event | `s2c_state_patch` | 更新目标状态 |
| Event | `s2c_tactical_prompt` | 与目标相关行动建议 |
| Store | `currentTeamObjective` | 团队目标 |

## 7. 状态与错误处理

- 目标文本过长时折叠显示。
- 个人目标权限错误时拒绝渲染并记录协议告警。
- 目标过期但未完成时显示“已失效”。
- AI KP 无法判断目标时保持上一目标，不随意清空。

## 8. 验收标准

- 手机行动面板始终有团队目标或等待提示。
- 个人目标只在本人手机可见。
- 没有个人目标的玩家界面不显得缺块。
- 目标变化进入日志和复盘。

## 9. 测试场景

1. 剧本无个人目标，只显示团队目标。
2. Player A 有个人目标，Player B 看不到。
3. Engine 更新目标为 completed，手机状态更新。
4. Host 大屏只显示团队目标。

## 10. 风险依赖

- 依赖 PDF 解析能抽取可用目标。
- 个人目标和防剧透策略必须统一。
- 玩家自带角色可能与个人目标适配较弱。
