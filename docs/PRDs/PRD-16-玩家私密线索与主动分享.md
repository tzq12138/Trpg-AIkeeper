# PRD-16 - 玩家私密线索与主动分享

**版本**: 1.0  
**状态**: 待开发  
**定位**: 私密信息接收、保存与团队分享

## 1. 背景

COC 和剧本杀体验依赖信息不对称。AI 自动 KP 必须默认保护私密线索，只给相关玩家手机。玩家可以线下口头讲，也可以通过手机主动分享，让 Engine 生成团队可见版本。

## 2. 目标

- 私密线索默认只在本人手机显示。
- 玩家可主动分享线索给团队。
- 分享后其他玩家和 Host 只看到团队可见版本。
- 分享行为进入事件日志，方便复盘。

## 3. 范围边界

**包含**: 私密线索通知、线索详情、`share_clue` 意图、公开化投影。  
**不包含**: 自动判断玩家线下是否说出口、玩家间私聊、线索拖拽编辑。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-16-1 | 作为玩家，我想独立查看只有我知道的线索。 | P0 |
| US-16-2 | 作为玩家，我想主动把某条线索分享给队友。 | P0 |
| US-16-3 | 作为其他玩家，我只应看到被分享后的团队版本。 | P0 |

## 5. 功能需求

1. `s2c_private_notice` 提醒玩家获得私密线索。
2. `s2c_state_patch` 将线索写入本人的 clues。
3. 线索详情页提供“分享给团队”按钮。
4. 点击分享提交 `share_clue`，params 包含 `clueId` 和可选玩家备注。
5. Engine 校验该玩家是否拥有该线索。
6. Engine 生成团队可见 `publicSummary`，不强制全文公开。
7. Host 可显示“某玩家分享了一条线索”的公共观察。
8. 分享后的团队线索进入其他玩家 clues 或 party clues。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| Intent | `share_clue` | 主动分享线索 |
| Event | `s2c_private_notice` | 私密提醒 |
| Event | `s2c_state_patch` | 私密/团队线索同步 |
| Event | `s2c_public_observation` | Host/队友公共观察 |
| Type | `PrivateClueShareRequest` | 分享请求 |

## 7. 状态与错误处理

- 玩家无该 clueId 时拒绝分享并提示状态已过期。
- 已分享线索再次分享时提示已公开，可补充备注。
- publicSummary 生成失败时使用线索安全摘要兜底。
- 私密线索未分享前不得进入 Host 快照。

## 8. 验收标准

- 私密线索默认不出现在其他玩家和 Host。
- 分享必须由玩家主动触发。
- 分享后团队看到的是公开版本。
- 分享事件可在个人档案中查询。

## 9. 测试场景

1. Player A 获得私密线索，B 无变化。
2. A 点击分享，B 收到团队线索。
3. A 分享不存在的 clueId，被拒绝。
4. Host 显示公共观察但不显示未公开全文。

## 10. 风险依赖

- 依赖 Engine 能生成安全公开摘要。
- 玩家线下口头泄露无法由系统阻止。
- 线索权限模型需要和防剧透 PRD-26 对齐。
