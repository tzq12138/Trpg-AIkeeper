# PRD-28 - 自动结局与可查询战役档案

**版本**: 1.0  
**状态**: 待开发  
**定位**: AI 自动判断结局，并生成可查询战役档案

## 1. 背景

AI 自动 KP 需要知道什么时候收束剧本。结局后不只给一篇固定总结，而要进入可查询档案：玩家想看线索、行动、检定、SAN/HP、回放片段都能查。

## 2. 目标

- AI KP 判断结局触发条件。
- 生成公共结局演出。
- 为每名玩家生成可查询档案索引。
- 保留完整战役日志供自由筛选。

## 3. 范围边界

**包含**: 结局判断、公共结局、个人档案入口、战役档案索引。  
**不包含**: 自动生成长篇小说、精美 PDF 导出、社交分享平台。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-28-1 | 作为玩家，我想知道这局达成了什么结局。 | P0 |
| US-28-2 | 作为玩家，我想自由查询自己关心的复盘内容。 | P0 |
| US-28-3 | 作为系统，我要把结局和关键证据关联起来。 | P1 |

## 5. 功能需求

1. AI KP 根据剧本结局条件、暴露度和玩家行动判断是否进入终局。
2. Engine 校验结局触发条件。
3. Host 播放公共结局事务。
4. Player 收到个人结局摘要和档案入口。
5. 战役档案按公共事件、个人事件、线索、检定、状态变化建立索引。
6. 玩家可按 PRD-20 自由查询。
7. 结局后房间进入只读/回放状态，除非从检查点恢复。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| Event | `s2c_campaign_ended` | 战役结束 |
| Event | `s2c_reveal_transaction` | 公共结局演出 |
| Type | `EndingReport` | 结局摘要 |
| Type | `PlayerArchiveQuery` | 自由查询 |
| Type | `EventLogEntry` | 档案来源 |

## 7. 状态与错误处理

- AI 建议结局但 Engine 校验未达成时继续游戏。
- 结局生成失败时保留当前状态并允许重试。
- 只读状态下拒绝新行动提交。
- 检查点恢复后退出 ended 状态。

## 8. 验收标准

- AI KP 可触发自动结局。
- 结局必须经过 Engine 校验。
- 玩家可进入个人档案自由查询。
- 结局后默认只读，可回放。

## 9. 测试场景

1. 达成核心真相和逃脱条件，触发结局。
2. AI 误判结局，Engine 拒绝。
3. 结局后玩家查询自己的 SAN 变化。
4. 从结局前检查点恢复继续。

## 10. 风险依赖

- 依赖剧本结构化出明确结局条件。
- 结局判断过早会破坏体验，需要 Engine 校验。
- 档案查询依赖 PRD-27 日志完整性。
