# PRD-12 - 玩家加入房间与手机准备流程

**版本**: 1.0  
**状态**: 待开发  
**定位**: AI 自动 KP 模式下的手机端入局流程

## 1. 背景

AI 自动 KP 模式下，玩家不能依赖人类 KP 逐个解释怎么开局。手机端必须让玩家自己完成加入房间、确认身份、导入角色、查看适配提醒并 ready。Host 大屏负责公共开场，手机负责个人准备状态。

## 2. 目标

- 让玩家通过扫码或房间码进入指定房间。
- 在手机竖屏上完成玩家名、角色卡、适配提醒和 ready。
- 让房主和 AI KP 能看到全员准备状态。
- 避免玩家未导入角色或未确认风险时误开局。

## 3. 范围边界

**包含**: 房间加入、玩家名、room token、本机身份保存、准备状态、ready 切换。  
**不包含**: 账号系统、远程语音聊天、角色创建器、剧本选择。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-12-1 | 作为玩家，我想扫码进入房间，不需要手动找链接。 | P0 |
| US-12-2 | 作为玩家，我想知道自己还差哪一步才能 ready。 | P0 |
| US-12-3 | 作为房主，我想看到哪些玩家已经准备好。 | P0 |

## 5. 功能需求

1. 玩家可通过二维码链接或输入房间码加入。
2. 首次加入时输入玩家名，并在本地保存 `roomToken`。
3. 手机准备页按步骤展示：加入房间、导入角色、查看适配、确认 ready。
4. 未完成角色导入时不能 ready。
5. 适配报告可有风险提醒，但不阻止 ready。
6. ready 状态通过 `ready_toggle` 意图提交给 Engine。
7. 所有人 ready 后，AI KP 可自动进入开场流程。
8. `POST /api/rooms/:roomId/join` 必须校验一次性邀请码或二维码签名，并对同一 IP 做速率限制，默认每分钟最多 5 次。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| REST | `POST /api/rooms/:roomId/join` | 加入房间并换取 room token |
| Intent | `ready_toggle` | 切换准备状态 |
| Event | `s2c_room_lobby_snapshot` | 同步大厅成员与 ready 状态 |
| Store | `PlayerOnboardingState` | 记录手机端准备步骤 |

## 7. 状态与错误处理

- 房间不存在时提示重新输入房间码。
- room token 失效时回到加入流程。
- 重复扫码同一房间时恢复本地身份。
- 游戏已开始后加入，进入旁观/等待补位状态，由房主有限授权决定。

## 8. 验收标准

- 玩家可仅用手机完成入房到 ready。
- 未导入角色不能 ready。
- 适配风险只提醒不拦截。
- 房主/Host 可看到 ready 进度。

## 9. 测试场景

1. 扫码进入房间，输入玩家名，获得 token。
2. 刷新页面后仍保持同一玩家身份。
3. 未导入角色点击 ready，被提示先导入角色。
4. 所有玩家 ready 后，房间进入 AI 开场。

## 10. 风险依赖

- 依赖房主开房流程提供二维码和房间码。
- 依赖角色导入 PRD-13 完成后才能 ready。
- 后续若做账号系统，需要迁移 room token 策略。
