# PRD-18 - 玩家澄清请求与误判纠正

**版本**: 1.0  
**状态**: 待开发  
**定位**: 玩家对 AI 听错、误解、误判的轻量纠错入口

## 1. 背景

AI 自动 KP 难免出现听错、误解行动、规则误判或上下文遗漏。玩家需要能提出澄清，但不能直接改 HP、SAN、线索等权威状态。纠错必须仍由 Engine 和 AI KP 处理。

## 2. 目标

- 玩家可对最近行动或结算发起澄清。
- AI KP 可解释、补问或请求 Engine 重算。
- 所有纠错过程进入事件日志。
- 防止玩家借纠错越权改状态。

## 3. 范围边界

**包含**: `clarification_request` 意图、最近行动引用、解释/重算结果。  
**不包含**: 玩家直接编辑状态、投票回滚、任意时间线分支。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-18-1 | 作为玩家，我想指出 AI KP 听错了我的行动。 | P0 |
| US-18-2 | 作为玩家，我想要求解释一次判定为什么这样结算。 | P0 |
| US-18-3 | 作为系统，我要防止纠错变成玩家改数值。 | P0 |

## 5. 功能需求

1. 最近行动回执提供“请求澄清”入口。
2. 澄清请求提交 `clarification_request`，包含 targetActionId、文本说明和可选证据。
3. AI KP 可返回解释、补问或重算建议。
4. Engine 只有在规则或输入确实错误时才生成修正事务。
5. 修正结果通过正常事件投影到 Host/Player。
6. 澄清请求有频率限制，避免打断游戏节奏。
7. 房主有限急救可查看澄清队列，但看不到剧透真相。
8. 默认澄清窗口为最近 5 分钟或最近 3 个 AI KP 回合，以先到边界为准；窗口时长可作为房间级配置项调整。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| Intent | `clarification_request` | 玩家澄清 |
| Event | `s2c_clarification_result` | 解释/补问/重算结果 |
| Event | `s2c_reveal_transaction` | 公共修正演出 |
| Type | `ActionReceipt` | 最近行动引用 |

## 7. 状态与错误处理

- targetActionId 不存在时提示无法定位行动。
- 已超过可澄清窗口时仅允许记录备注，不触发重算；默认窗口为最近 5 分钟或最近 3 个 AI KP 回合。
- 重算失败时给出解释，不改变状态。
- 恶意频繁请求触发冷却。

## 8. 验收标准

- 玩家能对最近行动发起澄清。
- 澄清不会直接修改权威状态。
- 合法重算通过 Engine 正常投影。
- 澄清和结果进入个人档案。

## 9. 测试场景

1. 玩家对最近 STT 文本发起“听错了”。
2. AI KP 返回解释，无状态改变。
3. Engine 发现规则误判，生成修正事务。
4. 超出窗口的请求被记录但不重算。

## 10. 风险依赖

- 依赖事件日志能定位原行动。
- 过多澄清会影响节奏，需要冷却和批处理。
- 重算策略需与检查点 PRD-27 保持边界。
