# PRD-20 - 玩家个人档案与自由复盘查询

**版本**: 1.0  
**状态**: 待开发  
**定位**: 局中/局后个人可查询档案

## 1. 背景

用户明确希望局后“想看啥都能查”，而不是只给固定报告。个人档案需要让玩家自由查看角色经历、线索、行动、检定、HP/SAN 变化、分享记录和公共回放片段。它既是局后复盘，也是局中查证工具。

## 2. 目标

- 建立玩家个人档案入口。
- 支持按类型筛选和搜索自己的事件。
- 关联公共回放片段和个人私密记录。
- 保护其他玩家私密信息。

## 3. 范围边界

**包含**: 个人行动、线索、检定、状态变化、分享、澄清、回放片段引用。  
**不包含**: 自由编辑战报、跨房间社交主页、导出精美 PDF。

## 4. 用户故事

| ID | 用户故事 | 优先级 |
|---|---|---|
| US-20-1 | 作为玩家，我想查自己发现过哪些线索。 | P0 |
| US-20-2 | 作为玩家，我想回看关键检定和 SAN 变化。 | P0 |
| US-20-3 | 作为玩家，我想按场景回到公共回放片段。 | P1 |

## 5. 功能需求

1. 玩家可进入“档案”标签。
2. 档案按线索、行动、检定、状态、分享、澄清、回放分类。
3. 支持关键词搜索和时间线排序。
4. 私密线索只对本人显示。
5. 公共事件可链接到 PRD-27 的回放时间点。
6. 局后档案保持可访问。
7. 自动结局后档案增加结局相关筛选。

## 6. 接口/事件依赖

| 类型 | 名称 | 用途 |
|---|---|---|
| REST | `GET /api/player/archive` | 查询个人档案 |
| Type | `PlayerArchiveQuery` | 查询条件 |
| Type | `EventLogEntry` | 档案来源 |
| Type | `Checkpoint` | 回放定位 |

## 7. 状态与错误处理

- 查询失败时提示重试，不清空本地已有档案。
- 无结果时给出空状态说明。
- 权限不匹配的事件不得返回。
- 长列表需要分页或虚拟滚动。

## 8. 验收标准

- 玩家可自由查询自己的核心经历。
- 私密内容不泄漏给其他玩家。
- 公共事件能跳转到回放片段。
- 局后仍可查看个人档案。

## 9. 测试场景

1. 按“线索”筛选，显示本人线索。
2. 搜索技能名，找到相关检定。
3. 点击公共事件，跳到回放片段。
4. Player B 无法查询 Player A 私密线索。

## 10. 风险依赖

- 依赖 PRD-27 事件日志质量。
- 长团数据量大，需要分页策略。
- 后续导出功能可独立扩展。
